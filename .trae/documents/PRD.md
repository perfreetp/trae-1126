## 1. 产品概述

港口装卸设备运维 Web 系统是面向码头设备部的综合性设备管理平台，旨在实现岸桥、场桥、牵引车和输送线等核心装卸设备的全生命周期管理。系统通过数字化手段提升设备运维效率，降低停机时间，保障港口作业的连续性和安全性。

- 核心目标：实现设备运维的数字化、可视化和智能化管理
- 目标用户：码头设备部管理人员、维修工程师、设备操作人员、司机

## 2. 核心功能

### 2.1 用户角色

| 角色 | 说明 | 核心权限 |
|------|------|----------|
| 管理员 | 设备部管理人员 | 全部功能权限、用户管理、系统配置 |
| 维修工程师 | 负责设备维修保养 | 故障处理、保养执行、备件申请 |
| 设备操作员 | 设备操作人员 | 设备状态查看、运行记录、异常反馈 |
| 司机 | 牵引车等设备驾驶员 | 运行记录、异常提交、油料登记 |

### 2.2 功能模块

1. **作业看板**：实时设备状态监控、泊位设备分布、作业班次管理
2. **设备台账**：设备基础信息管理、设备档案、设备调拨记录
3. **运行小时**：设备运行时长统计、班次运行记录、累计工时分析
4. **故障维修**：故障提报、维修派单、停机影响登记、外协维修跟踪、事故关联
5. **保养排程**：保养计划制定、保养窗口安排、安全检查项、保养执行记录
6. **油料轮胎**：油料消耗录入、轮胎更换记录、成本统计分析
7. **司机反馈**：司机异常提交、反馈处理跟踪、意见建议收集
8. **绩效统计**：设备完好率统计、维修成本对比、运维绩效分析

## 3. 核心流程

### 3.1 故障维修流程

司机发现设备异常 → 在司机反馈页面提交故障信息 → 系统自动生成故障工单 → 管理人员在故障维修页面派单给维修工程师 → 维修工程师现场处理，记录维修过程 → 如需停机，登记停机影响 → 维修完成后更新工单状态 → 系统自动统计完好率和维修成本

### 3.2 保养执行流程

系统根据保养计划自动生成保养任务 → 管理人员在保养排程页面安排保养窗口 → 保养前进行安全检查项确认 → 维修工程师执行保养，记录保养内容和更换备件 → 保养完成后更新设备状态 → 系统更新下次保养时间

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深海蓝 (#0F3460)，代表专业、可靠、工业感
- **辅助色**：活力橙 (#E94560) 用于强调和告警，科技青 (#16C79A) 用于成功状态
- **中性色**：深灰 (#1A1A2E)、中灰 (#535461)、浅灰 (#F5F5F5)
- **按钮风格**：直角微圆角 (4px)，工业风格，带有轻微阴影
- **字体**：标题使用 "Noto Sans SC" 粗体，正文使用 "Noto Sans SC" 常规
- **布局风格**：顶部导航 + 侧边栏菜单 + 内容区域，卡片式布局，数据表格为主
- **图标风格**：使用 lucide-react 线性图标，简洁清晰

### 4.2 页面设计概述

| 页面名称 | 模块名称 | UI 元素 |
|-----------|-------------|-------------|
| 作业看板 | 整体布局 | 顶部状态栏、左侧设备分类、中间泊位分布图、右侧告警列表 |
| 作业看板 | 泊位分布 | 网格布局展示泊位，不同颜色标识设备状态，悬停显示详情 |
| 设备台账 | 设备列表 | 数据表格，支持筛选、排序、分页，操作列包含查看/编辑/调拨 |
| 故障维修 | 工单列表 | 卡片式列表，显示工单状态、优先级、处理人，支持拖拽排序 |
| 绩效统计 | 数据看板 | 多种图表组合（柱状图、折线图、饼图），关键指标卡片 |

### 4.3 响应式设计

- 采用桌面优先设计，针对 1920px 宽屏优化
- 支持 1366px 及以上分辨率自适应
- 侧边栏可折叠，增加内容区域空间
- 数据表格在小屏幕下支持横向滚动

### 4.4 交互动效

- 页面加载时数据区域渐入显示
- 状态变化时有平滑过渡动画
- 卡片悬停时有轻微上浮效果
- 告警信息有呼吸灯动画提醒
